Introduction

Multiplicative structure often shows up in number theory when one considers common arithmetic functions (functions whose domain are the integers).

Definition. We say an arithmetic function f:Z→R f: \Z \to \R f:Z→R is multiplicative if it satisfies f(xy)=f(x)f(y)\begin{aligned} f(xy) = f(x) f(y) \end{aligned}f(xy)=f(x)f(y)​ for all coprime integers x,y x, y x,y, that is, gcd⁡(x,y)=1 \gcd(x, y) = 1 gcd(x,y)=1. An example is Euler's totient function φ(n) \varphi(n) φ(n) which is defined as the number of integers between 1 1 1 and n n n relatively prime to n n n.

In addition, we say that f f f is completely multiplicative if it satisfies (1) for all integers x,y x, y x,y, not just those which are relatively prime. From now on we will use the term "multiplicative" to refer to completely multiplicative functions and take our domain to be the reals.

In contrast, some functions have additive structure. For example, for any real x,y x, y x,y, the line f(x)=kx f(x) = k xf(x)=kx for some fixed constant k k k satisfies

f(x+y)=f(x)+f(y)\begin{aligned} f(x + y) = f(x) + f(y) \end{aligned}f(x+y)=f(x)+f(y)​

since f(x+y)=k(x+y)=kx+ky=f(x)+f(y) f(x + y) = k(x + y) = k x + k y = f(x) + f(y) f(x+y)=k(x+y)=kx+ky=f(x)+f(y).

Examples

Some functions turn multiplicative structure into additive structure; for example, for any real x,y>0 x, y > 0 x,y>0, the logarithm function (base irrelevant) satisfies

log⁡(xy)=log⁡(x)+log⁡(y)\begin{aligned} \log(xy) = \log(x) + \log(y) \end{aligned}log(xy)=log(x)+log(y)​

which is an interesting and often quite useful property turning multiplications into additions. However, there is no easy formula (that I know of) for log⁡(x+y) \log(x + y) log(x+y), which is quite unfortunate.

Every symmetric positive-definite (s.p.d.) matrix Θ \Theta Θ has a lower triangular Cholesky factor L L L satisfying Θ=LL⊤ \Theta = L L^{\top} Θ=LL⊤. In some sense all matrix factorizations are product structures since they write a complicated matrix as the product of matrices that are hopefully esimpler or easier to work with. Expanding LL⊤ L L^{\top} LL⊤ reveals the hidden additive structure of the Cholesky factorization, owing to the fact that the lower triangularity of L L L implies a sort of decreasing or nested structure. Indeed, we can write Θ \Theta Θ as the contributions of rank-one matrices given by the outer product of the columns of L L L (4) (which is generally true for any matrix product) where each outer product affects smaller and smaller submatrices of Θ \Theta Θ (from the triangularity of L L L).

Θ=LL⊤=L:,1L:,1⊤+⋯+L:,nL:,n⊤\begin{aligned} \Theta = L L^{\top} = L_{:, 1} L_{:, 1}^{\top} + \dotsb + L_{:, n} L_{:, n}^{\top} \end{aligned}Θ=LL⊤=L:,1​L:,1⊤​+⋯+L:,n​L:,n⊤​​

Although the Cholesky factor is both an additive and multiplicative factorization, the chol⁡(⋅) \operatorname{chol}(\cdot) chol(⋅) operator itself does not play well with addition or multiplication. For multiplication the product of two s.p.d. matrices is not even necessarily s.p.d. and for addition even adding a multiple of the identity, that is, trying to factor Θ+σ2Id \Theta + \sigma^2 \text{Id} Θ+σ2Id requires somewhat sophisticated tricks [3], even if one is able to factor Θ \Theta Θ efficiently.

Finally, the trace⁡(⋅) \operatorname{trace}(\cdot) trace(⋅) and det⁡(⋅) \det(\cdot) det(⋅) operators have additive and multiplicative structure, respectively. That is, for all square matrices A,B A, B A,B of the same size,

trace⁡(A+B)=trace⁡(A)+trace⁡(B)det⁡(AB)=det⁡(A)det⁡(B)\begin{aligned} \operatorname{trace}(A + B) &= \operatorname{trace}(A) + \operatorname{trace}(B) \\ \det(AB) &= \det(A) \det(B) \end{aligned}trace(A+B)det(AB)​=trace(A)+trace(B)=det(A)det(B)​

However, there are not as nice decompositions for trace⁡(AB) \operatorname{trace}(AB) trace(AB) or det⁡(A+B) \det(A + B) det(A+B). Although it is possible to turn trace⁡(AB) \operatorname{trace}(AB) trace(AB) into additive structure by expanding the matrix-matrix product and to turn det⁡(A+B) \det(A + B) det(A+B) into multiplicative structure by the Sherman–Morrison–Woodbury identity, this is more of a conversion and a bit less natural than applying (5) directly.

Problem statement

Throughout these examples it seems although multiplicative and additive structure can be converted between, they are hard to mix in some sense. In order to make this notion precise,

Definition. Suppose some function T:R→R T: \R \to \R T:R→R satisfies both T(x+y)T(xy)=T(x)T(y)T(x)+T(y)[multiplicative structure]\begin{aligned} \hphantom{T(x + y)} \mathllap{T(x y)} &= \mathrlap{T(x) T(y)} \hphantom{T(x) + T(y)} && \text{[multiplicative structure]} \end{aligned}T(x+y)T(xy)​=T(x)T(y)T(x)+T(y)​​[multiplicative structure]​ T(x+y)=T(x)+T(y)[multiplicative structure][additive structure]\begin{aligned} T(x + y) &= T(x) + T(y) && \hphantom{\text{[multiplicative structure]}} \mathllap{\text{[additive structure]}} \end{aligned}T(x+y)​=T(x)+T(y)​​[multiplicative structure][additive structure]​ for all x,y∈R x, y \in \R x,y∈R. What can we infer about T T T just from these conditions?

Solution

Our strategy will to identify a few specific observations and then work our way up to the naturals N \mathbb{N} N, the integers Z \Z Z, the rationals Q \mathbb{Q} Q, and then finally, the reals R \R R. Extension to the complex numbers C \mathbb{C} C is left as an exercise for the reader.

Naturals

First, we observe that if we fix y=1 y = 1 y=1, then the multiplicative condition (6) implies ∀x∈R \forall x \in \R ∀x∈R

T(1⋅x)=T(1)⋅T(x)=T(x)\begin{aligned} T(1 \cdot x) &= T(1) \cdot T(x) = T(x) \\ \end{aligned}T(1⋅x)​=T(1)⋅T(x)=T(x)​

Moving T(x) T(x) T(x) to the left side and factoring, we have

T(1⋅x)=T(1)⋅T(x)(T(1)−1)T(x)=0T(x)\begin{aligned} \hphantom{T(1 \cdot x) = T(1) \cdot T(x)} \mathllap{(T(1) - 1) T(x)} &= \mathrlap{0} \hphantom{T(x)} \end{aligned}T(1⋅x)=T(1)⋅T(x)(T(1)−1)T(x)​=0T(x)​

implying either T(1)=1 T(1) = 1 T(1)=1 or T(x)=0 T(x) = 0 T(x)=0 for all x x x. So immediately the constant function T(x)=0 T(x) = 0 T(x)=0 is a candidate solution and indeed, it also satisfies the additive condition (7) trivially. For the more interesting situation where T(1)=1 T(1) = 1 T(1)=1, now using the additive condition (7), we have

T(x+1)=T(x)+T(1)=T(x)+1\begin{aligned} T(x + 1) &= T(x) + T(1) = T(x) + 1 \\ \end{aligned}T(x+1)​=T(x)+T(1)=T(x)+1​

Therefore T(2)=T(1)+1=2 T(2) = T(1) + 1 = 2 T(2)=T(1)+1=2, T(3)=T(2)+1=3 T(3) = T(2) + 1 = 3 T(3)=T(2)+1=3, and so on. By induction T(x)=x T(x) = x T(x)=x for all natural numbers x x x (where we exclude 0 as a natural number for now).

Integers

To extend this result to the integers, first we fill in the hole at 0 by making use of (7),

T(0+0)=2T(0)=T(0)  ⟹  T(0)=0\begin{aligned} T(0 + 0) &= 2 T(0) = T(0) \\ \implies T(0) &= 0 \end{aligned}T(0+0)⟹T(0)​=2T(0)=T(0)=0​

Then we observe for a natural number x x x and a negative number y=−x y = -x y=−x, using (7),

T(x+(−x))=T(x)+T(−x)=T(0)=0  ⟹  T(−x)=−T(x)\begin{aligned} T(x + (-x)) &= T(x) + T(-x) = T(0) = 0 \\ \implies T(-x) &= -T(x) \end{aligned}T(x+(−x))⟹T(−x)​=T(x)+T(−x)=T(0)=0=−T(x)​

where we use that T(x)=x T(x) = x T(x)=x for natural x x x, concluding that T(x)=x T(x) = x T(x)=x for integer x x x.

Rationals

To extend this result to the rationals: let x=p/q x = p/q x=p/q for integers p,q p, q p,q and let y=q y = q y=q. From (6),

T((p/q)q)=T(p/q)T(q)=T(p)=p  ⟹  T(p/q)=p/T(q)=p/q\begin{aligned} T((p/q) q) &= T(p/q) T(q) = T(p) = p \\ \implies T(p/q) &= p/T(q) = p/q \end{aligned}T((p/q)q)⟹T(p/q)​=T(p/q)T(q)=T(p)=p=p/T(q)=p/q​

where we use T(p)=p T(p) = p T(p)=p and T(q)=q T(q) = q T(q)=q for integers p,q p, q p,q, concluding that T(x)=x T(x) = x T(x)=x for rational x x x.

Reals

Throughout all these examples it seems that T(x)=x T(x) = x T(x)=x, but it is hard to extend to the reals without additional structure; we've sort of hit the limit on what our assumptions tell us. We would like our function to commute with limits, that is, we would like for any sequence (xn) (x_n) (xn​),

T(lim⁡n→∞xn)=lim⁡n→∞T(xn)\begin{aligned} T \left (\lim_{n \to \infty} x_n \right ) &= \lim_{n \to \infty} T(x_n) \end{aligned}T(n→∞lim​xn​)​=n→∞lim​T(xn​)​

If we enforce that T(x) T(x) T(x) must be continuous, then every real number x x x is a Cauchy sequence of rational numbers, so assume (xn)→x (x_n) \to x (xn​)→x for rational xn x_n xn​ and observe that

T(x)=T(lim⁡n→∞xn)=lim⁡n→∞T(xn)=lim⁡n→∞xn=x\begin{aligned} T(x) = T \left (\lim_{n \to \infty} x_n \right) = \lim_{n \to \infty} T(x_n) = \lim_{n \to \infty} x_n = x \end{aligned}T(x)=T(n→∞lim​xn​)=n→∞lim​T(xn​)=n→∞lim​xn​=x​

Theorem. Suppose some continuous function T:R→R T: \R \to \R T:R→R satisfies both

T(xy)=T(x)T(y)[multiplicative structure]T(x+y)=T(x)+T(y)[additive structure]\begin{aligned} T(x y) &= T(x) T(y) && \text{[multiplicative structure]} \\ T(x + y) &= T(x) + T(y) && \text{[additive structure]} \end{aligned}T(xy)T(x+y)​=T(x)T(y)=T(x)+T(y)​​[multiplicative structure][additive structure]​

for all x,y∈R x, y \in \R x,y∈R. Then either

  1. T(x)=0 T(x) = 0 T(x)=0 for all x∈R x \in \R x∈R or

  2. T(x)=x T(x) = x T(x)=x for all x∈R x \in \R x∈R.

Conclusion
